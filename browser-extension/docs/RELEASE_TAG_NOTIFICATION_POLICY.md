# Release Tag Notification Policy

## Trigger
Release notifications are sent only when a version tag matching `v*` is pushed. Ordinary build successes do not trigger notifications. The notification is gated on the release job completing successfully.

## Channels
Notifications are delivered via Telegram. The message includes the release version, change log summary, download links, and the release actor. The notification is generated by `scripts/telegram-release-notify.py` and prepared by `tools/prepare-release-notes.mjs`.

## Content
The notification contains the tag name, a link to the GitHub release, the change log extracted from `RELEASE_NOTES.md`, and download links from `DOWNLOADS.md`. All content is published through the Telegram bot API with HTML formatting and web page preview disabled.

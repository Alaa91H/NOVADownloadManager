# Release

This project exposes the same high-level release contract as `Alaa91H/NOVA-Browser-Extension` while keeping WXT as the internal compiler.

## Local release build

```bash
corepack enable
pnpm install
npm run build:zip
pnpm verify:release:reuse-build
```

Equivalent explicit command:

```bash
node scripts/run-python.js build.py --clean --zip --version vX.Y.Z
node tools/run-release-checks.js --package-only
```

## GitHub release

Push a version tag. The extension manifest/package version is derived from this tag automatically:

```bash
git tag vX.Y.Z
git push origin vX.Y.Z
```

Prerelease tags are supported for CI and GitHub release builds:

```bash
git tag vX.Y.Z-beta.N
git push origin vX.Y.Z-beta.N
```

Because browser manifests require numeric versions, a tag such as `v1.0.0-beta.15` is written into generated manifests as `1.0.0.15`.

The workflow publishes files staged under `dist/release-assets`.

## Outputs

- `dist/chromium/` unpacked Chromium/Chrome extension.
- `dist/edge/` unpacked Microsoft Edge extension.
- `dist/firefox/` unpacked Firefox extension.
- `dist/packages/*-chrome-*.zip` or `.crx` Chrome package.
- `dist/packages/*-edge-*.zip` Edge package.
- `dist/packages/*-firefox-*.xpi` Firefox package.
- `dist/packages/SHA256SUMS.txt` checksums.
- `dist/packages/release-manifest.json` release metadata.
- `dist/release-assets/CHANGELOG.md` generated changelog.
- Generated GitHub Release notes with download links and changelog.

## Artifact policy

Release artifact files are generated by the release pipeline and staged under `dist/release-assets`. A release must not be published from an unverified local build artifact.
